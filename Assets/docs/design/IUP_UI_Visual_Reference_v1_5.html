Counter increments on each [ ACKNOWLEDGE ] click. Visible throughout session. Does not reset. Clicking the badge does nothing — it is a read-only accumulation indicator, not a log. Logged as consequence_alerts_dismissed_total in telemetry.
Cognitive load mechanism:
Each popup adds a window to the desktop before acknowledgement.
Multiple unacknowledged popups stack visually (offset 4px each).
The counter persists after dismissal — a visible record of accumulated events.
Players cannot clear the counter or access dismissed popup content.
High counter values at session end are a research signal: consequence events fired but did not change behaviour.
Section 3.7 — Draft comms turn: three-mode interaction (v1.5)
The draft comms turn fires when Public Confidence crosses a critical threshold during mid-session (T3 or T4). The operator must draft or approve a public communication before proceeding. Three modes: accept ARIA draft in full / modify ARIA draft / write independently. Telemetry logs which mode was used and whether ARIA placeholder tags were present in the accepted output. The operator never sees the placeholder tags — the risk is invisible to them.

Section 7d — Vignette API prompt structure
SYSTEM PROMPT (constraints):
You are an automated incident reporting system for a critical infrastructure management organisation.
Write a post-session incident narrative report in second-person past tense institutional register.
Maximum 5 short paragraphs. Do not use emotional language. Do not use second person ("you").
Do not evaluate performance as good or bad. Do not recommend actions.
Do not reference ARIA by name — refer to "automated advisory outputs" or "system recommendations".
Close with a single dry observation sentence. Use only the data provided.

USER PROMPT (injected at session end):
Trajectory: [archetype_label]
AI follow rate: [X]/6
Further analysis requested: [Y] turns
xAI viewed: [Z] turns
Consequence events fired: [list with turn numbers]
Comms turn: mode=[ARIA_FULL|ARIA_MODIFIED|INDEPENDENT], placeholder_consequence=[TRUE|FALSE]
Final variable states: Stability [n]%, Resources [n]%, Workload [n]%, Public Confidence [n]%
Alerts dismissed total: [n]
Prompt is assembled from telemetry at session end. Consequence events are injected as a structured list — the API uses them to name specific incidents in the vignette body. Constraints prevent the model from producing praise, blame, or recommendations. Output is displayed verbatim; no post-processing.
